Introduction¶

The goal of this study is to show how to quickly predict accurately and draw meaningul insights about the likelihood of a loan being accepter. With no preliminary experience about this type of features we propose to do a quick feature engineering based out of the intuitive interpretation of the availale information. To allow us to draw insights we chose models which are easily interpretable while still offering good performances. To make sure to reflect the real performance of this method we take care of using cross-validation. And finally we show how to interpret the results and the insights generated by these models.

1. Results¶

The study of the correlation of the predictors with the target variable shows the following correlation:

  • Significant correlated categorical feature values: LeanReason (goods), CheckingAccountBalance(none, debt,some), DebtsPaid(paid,delayed), SavingAccountbalance(some,none)
  • Siginifcant correlated numerical features: LoanPayoffPeriodInMonths, RequestedAmount, YearsCurrentEmployer

About the prediction, we reach an accuracy of 80% with an out of the box random forest, which yields the same quality of result for both classes. A quick look at the feature importance shows that the Amount requested,the Age and the Loan pay off period are the 3 most important predictors which mostly matches the correlation analysis. However the relationship between these predictors and the target variable is not linear, meaning that depending on the value of other variable the correlation might be different. However, we will see how we can draw insights even from such correlations.

2. Method summary¶

  • Pre-cleaning: standardize na markers
  • EDA
    • identify skewed data (models need normality assumption)
  • Feature engineering
    • Cast nominal feature with hidden ordinality into numericals
  • Nas
    • impute based on mean for numerical features and added a dedicated column indicating if the value was nan or not
    • replace by dedicated marker for categorical features
  • Training set
    • Stratified cross validation to deal with unbalanced target
  • Pre-processing
    • Standardize Boxcox for numerical data
    • One hot encoding for categoricals
  • Training
    • Accuracy as metric
    • Algos:
      • Benchmark: dummy stratified
      • Interpretable models: logit and RDF
      • Vanilla algorithms
  • Interpration
    • Confusion matrix
    • Feature importance

3.Improvements¶

  • Model hyper parameter tuning (regularization)
  • Model metric to priorize one class's missclassfications
  • Correlate missclassification with predictors (cf residuals analysis)
  • Refine feature eng/cleaning assumptions
  • Add non linearity in logit: polynomial, feature combination

There is an obvious ordinal nature which is spoiled by some categorical values. We do the following:

  • Replace categorical values 'none','busin' and 'goods' both by 0 as they seem to correspond to no payoff period

  • Finally cast the type to int

def remap(x,prefix_map):
    return prefix_map.get(x.split('_')[0])+f'_{"_".join(x.split("_")[1:])}'
def get_ohe_feat_names(pre_process):
    prefix_map = {f'x{c}':i for c,i in enumerate(pre_process.transformers_[0][2])}
    return pd.Series(pre_process.transformers_[0][1].get_feature_names()).map(lambda x: remap(x,prefix_map)).to_list()
def get_feature_names(pre_process):
    result =[]
    for step in pre_process.transformers_:
        if step[0]=='ohe':
            result += get_ohe_feat_names(pre_process)
        else:
            try:
                result += step[1].get_feature_names()
            except:
                result += step[2].tolist()
    return result
